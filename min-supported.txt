# changeguard/min-supported.txt
#
# 이 SHA보다 오래된 정책 SHA에 pin된 소비자는 ci-caller.py가 다음 워크플로우 실행에서
# deprecation ALERT 발송 + 워크플로우 실패 처리한다. 이 파일은 워크플로우 실행 시점에
# upstream main에서 동적으로 fetch되며, 소비자가 ci-caller.py v0.10 이상에 도달했을 때
# 활성화된다.
#
# 형식: 첫 번째로 매치되는 40자 hex SHA를 floor로 사용. `#` 시작 라인·빈 줄은 무시.
#
# 보안 모델:
#   - 정책 *코드*는 여전히 SHA-pinned (mutable upstream 안 함, RCE 차단)
#   - 이 floor만 동적 — 위험 표면은 갱신 강제(DoS) 한정
#   - 네트워크 실패시 fail-open (소비자 게이트가 멈추지 않도록)
#
# 갱신 시점: 특정 정책 SHA에서 결함·취약이 확정됐을 때 그 다음 SHA를 floor로 올린다.
# 이 갱신 자체도 PR + CODEOWNERS 승인 + pre-merge-review 게이트를 거친다.
#
# 초기값: 최초 commit (1cf41e88) — permissive baseline. 현재 deprecated된 정책 없음.
